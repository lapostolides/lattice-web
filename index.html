What makes it different

Three foundations

01

Physics-first validation

A deterministic 7-step physics pipeline (units, ranges, equations, CFL stability, mesh quality, paradigm rules) runs before any experiment executes — no LLM in the validation loop.

02

Knowledge graph + CRB feasibility

A Neo4j sensor/algorithm knowledge graph stores Cramér-Rao bounds and cross-family physics relationships. Infeasible designs are filtered before they ever burn a GPU cycle.

03

Physics capability algebra

Sensors decompose into composable Fisher Information primitives. The system reasons about capabilities — not products — enabling novel sensor discovery from first principles.

How it works

The autonomous loop

Hypothesis → Scene → Physics-validate → Execute → Analyze. Iterate until the analyst agent says stop.

  1. 1

    Hypothesis

    A researcher states a downstream-task hypothesis. The agent formalizes it.

  2. 2

    Scene

    A scene-generation agent composes geometry, materials, sensor, and lighting.

  3. 3

    Physics-validate

    Deterministic checks reject impossible scenes. No LLM in the validation path.

  4. 4

    Execute & Analyze

    Run the simulation, evaluate against the hypothesis, decide whether to iterate.
